=======================================================
                  Node.js 学习笔记
=======================================================

一、核心知识点

1. Node.js基础
   - 运行环境: V8引擎，服务器端JavaScript
   - 特点: 异步非阻塞I/O、事件驱动、单线程
   - 全局对象: global、process、console

2. NPM基础
   - 安装与配置: npm install、npm config set registry
   - package.json: 项目配置文件
   - 依赖管理: dependencies、devDependencies
   - 脚本命令: scripts字段

3. 常用模块
   - fs: 文件系统操作
   - path: 路径处理
   - http: HTTP服务器/客户端
   - url: URL解析
   - querystring: 查询字符串处理
   - child_process: 子进程

4. 文件系统操作
   - 同步操作: fs.readFileSync、fs.writeFileSync
   - 异步操作: fs.readFile、fs.writeFile
   - 目录操作: fs.mkdir、fs.rmdir、fs.readdir

5. HTTP模块
   - 创建服务器: http.createServer()
   - 请求对象: req（请求信息）
   - 响应对象: res（响应信息）
   - 状态码: 200、404、500等

6. 模块化
   - CommonJS: module.exports、require()
   - ES Modules: export、import（需要配置）

7. 异步编程
   - 回调函数
   - Promise
   - async/await

二、重点难点

1. 异步非阻塞机制
   - 事件循环的工作原理
   - 宏任务与微任务
   - 避免回调地狱

2. 模块加载机制
   - CommonJS vs ES Modules
   - 循环依赖问题
   - 模块缓存机制

3. 文件操作的错误处理
   - 异步操作的错误处理
   - 权限问题
   - 路径处理

三、学习心得

- Node.js让JavaScript可以运行在服务器端
- 异步编程是Node.js的核心，需要适应回调思维
- 模块化让代码组织更清晰
- NPM生态丰富，需要合理管理依赖

四、关键概念总结

- 事件循环: Node.js处理异步操作的机制
- 单线程: Node.js主线程是单线程，但I/O操作是异步的
- 模块缓存: 模块第一次加载后会被缓存，后续require返回同一实例
- 进程管理: 通过child_process或cluster模块实现多进程
