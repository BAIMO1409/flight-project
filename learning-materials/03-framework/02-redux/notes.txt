=======================================================
                  Redux 学习笔记
=======================================================

一、核心知识点

1. Redux基础
   - 简介: 用于JavaScript应用的状态管理库
   - 特点: 单一数据源、状态只读、纯函数修改
   - 安装: npm install redux react-redux

2. 三大原则
   - 单一数据源: 整个应用的状态存储在一个Store中
   - 状态只读: 唯一改变状态的方式是触发Action
   - 纯函数修改: Reducer是纯函数，接收旧状态和Action，返回新状态

3. Action
   - 简介: 描述状态变化的对象
   - 结构: { type: 'ACTION_TYPE', payload: data }
   - Action创建函数: 返回Action对象的函数

4. Reducer
   - 简介: 纯函数，处理状态变化
   - 结构: (state, action) => newState
   - 特点: 无副作用、相同输入相同输出

5. Store
   - 简介: 状态容器
   - 创建: createStore(reducer)
   - 方法: getState()、dispatch(action)、subscribe(listener)

6. 中间件（Middleware）
   - 作用: 扩展dispatch功能
   - 常用中间件: redux-thunk（异步操作）、redux-saga、redux-logger

7. React-Redux
   - Provider: 提供Store给所有组件
   - connect: 连接组件与Store
   - useSelector: 选择Store中的状态
   - useDispatch: 获取dispatch函数

8. Redux Toolkit
   - configureStore: 创建Store
   - createSlice: 创建Reducer和Action
   - createAsyncThunk: 创建异步Action

二、重点难点

1. 异步操作处理
   - 使用redux-thunk处理异步
   - 使用createAsyncThunk（Redux Toolkit）
   - 三种状态: pending、fulfilled、rejected

2. 状态结构设计
   - 扁平化设计
   - 规范化数据结构（类似数据库）
   - 避免嵌套过深

3. 性能优化
   - useSelector的依赖比较
   - React.memo优化组件渲染
   - Reselect创建记忆化选择器

三、学习心得

- Redux适合大型应用的状态管理
- 单一数据源让状态追踪更简单
- Redux Toolkit大幅简化了Redux的使用
- 中间件机制让异步操作更可控

四、关键概念总结

- Action: 状态变化的描述
- Reducer: 状态变化的处理函数
- Store: 状态容器
- Dispatch: 触发Action的方法
- Selector: 从Store中选择数据的函数
